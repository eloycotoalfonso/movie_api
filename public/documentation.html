My movies DOCUMENTATION

This is an server side component aplication for movies. Users will be able to log in, sign up, and get different information about movies, directors and so on. They will also be able to update their personal information and create a list of their favorite movies.
This task is based on the server side, the client-side will be part of the next achievement of the course.

Request URL HTTP Method Request body data format Response body data format
Return a list of ALL movies to the user /movies GET None A JSON object that retrieves data about all the films
Return data about a single movie by title /movies/[Title] GET None A JSON Object retrieving data about a single film
Return data about a genre by name/title /movies/genre/[Genre] GET None A JSON Object retrieving data about one particular genre
Return data about a director /movies/director/[Name] GET None A JSON Object retrieving data about one particular director
Allow new users to register /users POST A JSON with the following format
{
"username": String,
"password": String,
"email": String,
"birthday": Date
}
For extample: { "username": "123", "password": "123", "email": "123@123.com", "birthday": "1234-12-12" } 		A text messsage indicating the user has been registered, or a text message indicating the user is already used if applies.
Allow users to update their user info (username) /users/[Name] PUT A JSON{
username: String, (required)
password: String, (required)
email: String, (required)
} For example: { "username": "12345", "password": "12345", "email": "11111" } 		A JSON with the updated information of the user
Allow users to add a movie to their list of favorites /users/[name]/movies/[movieID] POST None A JSON with the updated information of the user
Allow users to remove a movie from their list of favorites /users/[name]/movies/[movieID] DELETE None A text message indicating that the movie was not found in the user's favorites, or a JSON wih the updated information of the user.
Allow existing users to deregister /users/[Name] DELETE None A text message confirming that the user was successfully removed.
Allow users to log in /login POST Parameters of "Username" and "Password" A JSON with two pair key value parameters, one is a JSON wih the user information and the second one is the user token
